会话跟踪技术：客户端第一次发请求给客户端，服务器获取session，获取不到则创建新的，然后响应给客户端；下次客户端发请求的时候会把这个 sessionID 带给服务器，那么服务器就能获取到了，这样就能判断这个客户端和上次是同一个客户端。常用API：request.getSession() -> 获取当前的会话，没有则创建一个新的；request.getSeesion(true) -> 效果和不带参数相同； request.getSession(false) -> 获取当前会话，没有则返回null，不会创建新的； session.getId() -> 获取sessionID； session有效的时间是半个小时； session.invalidate() -> 强制让会话失效； session的作用域就是：其作用域就是一次会话，如果换一个浏览器（比如从chrome换成ie）那么再次请求的时候就获取不到这个上一个浏览器保存在作用域中的 key 和 value 了；
