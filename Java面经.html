1.Autowired和resource的区别？默认按什么注入？

  1. @Autowired：

    • @Autowired 注解是 Spring 的原生注解，用于自动注入依赖关系。
    • 默认情况下，它按类型（Type）进行自动装配。如果找到多个相同类型的 bean，则会抛出异常，除非你进一步限定选择。
    • @Autowired 可以与 @Qualifier 注解一起使用，以指定按名称装配的 bean，从而解决相同类型多个 bean 的冲突问题。
    • 它可以用在构造函数、字段（属性）、设置方法上。
    • 从 Spring 4.3 版本开始，如果一个类只定义了一个构造函数，那么使用 @Autowired 注解可以是可选的。

  2. @Resource：

    • @Resource 注解是由 JSR-250 规范提供的，因此它不仅限于 Spring，也可以用在 Java EE 中。
    • @Resource 默认按名称（Name）进行自动装配。如果没有指定名称，它会回退到按类型匹配。
    • @Resource 有一个 name 属性，允许显式地指定要注入的 bean 的名称。
    • 它主要用于字段注入和设置方法注入，不推荐用于构造函数注入。

默认注入方式的对比：

  • @Autowired 默认是按类型（Type）注入。
  • @Resource 默认是按名称（Name）注入，如果没有指定名称，则按类型（Type）注入。

（3）怎样避免死锁

概念：就是两个或者多个线程相互等待对方释放资源的情况下，导致所有的线程无法继续执行的情况。

四个条件：互斥、请求与保持、不可剥夺、循环等待

避免循环等待：确保线程按相同的顺序获取锁，避免循环等待的可能

使用超时机制：设置获取锁的超时时间、等待一段时间后放弃获取

使用可重入锁：允许同一个线程多次获取同一个锁，这样这个线程在获取了锁A的情况下，使用别的方法再次获取这个锁A的时候依然可以获取，不会导致死锁；

（4）concurrentHashMap

为了解决HashMap线程不安全的问题，java提供了线程安全的HashMap实现类ConcurrentHashMap。它的内部采用了分段锁，将整个Map拆分成多个小的HashMap，每个小的HashMap就都有自己的锁，不同的线程可以同时访问不同的小Map，从而实现了线程安全。在进行插入的、删除和扩容等操作的时候，只需要锁住当前小Map，不会对整个Map进行锁定，提高了并发访问的效率。

（11） http的状态码

  • 信息性状态码（1xx）： 100，表示服务器收到请求的初始部分，可以继续发送请求的剩余部分；101：服务器已经理解服务器的请求，会切换到不同的协议；
  • 成功状态码（2xx）：200： 请求已成功，服务器正常处理并返回所请求的资源；201：请求已成功，并在服务器上创建了新的资源；202：请求已成功，但响应不包含实体主体部分；
  • 重定向状态码（3xx）：301：请求的资源已永久移动到新位置；302：请求的资源暂时移动到一个不同的位置；304：客户端已经可以使用缓存的副本，因为资源未被修改；
  • 客户端错误状态码（4xx）：400：请求无效，服务器无法理解；401：请求要求身份验证；404：请求的资源不存在；
  • 服务器错误状态码（5xx）：500：服务器遇到了意外情况，无法完成请求。503：服务器当前无法处理请求，一段时间后可能恢复正常；

（14）图片过大怎么处理

分片处理时一种将大文件分割成较小的块（片段）进行上传或传输的方法。在图片上传的场景中，当图片文件较大时，可以将其分割成多个较小的片段进行逐个上传，而不是一次性将整个文件上传。优点：

  • 提高上传效率，当中间发生中断或失败，只需要重新上传受影响的片段。
  • 减轻服务器负载，处理小文件不会占用大量的系统资源和网络带宽。
  • 提高上传的可靠性：分片处理中，每个片段都可以进行独立的校验和验证；
